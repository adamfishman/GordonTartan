## Project Overview

GordonTartan is a fork of [Tartanify.com](https://tartanify.com) вЂ” a collection of 5,000+ tartan patterns from the Scottish Register of Tartans. Users can browse, search, download (SVG/PNG), and create custom tartan variants.

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Astro v5 (SSR / server output mode) |
| Interactive UI | React 19 (Astro island components) |
| Language | TypeScript (strict mode) |
| Runtime | Cloudflare Workers |
| Database | Cloudflare D1 (SQLite-compatible) |
| Search | Lunr.js (pre-built index at build time) |
| Local dev | Wrangler (`wrangler dev`) |
| Testing | Vitest (node environment) |

## Development Commands

```bash
npm install          # Install dependencies
npm run dev          # Local dev server via wrangler dev (includes D1 proxy)
npm test             # Run Vitest tests (one-shot)
npm run test:watch   # Run Vitest in watch mode
npm run build        # Build: generates search index в†’ astro build
npm run seed         # Seed remote D1 from CSV (requires wrangler login)
npm run seed -- --local  # Seed local D1 (used with wrangler dev)
npm run deploy       # Build + wrangler deploy to Cloudflare
```

The `build` script runs two steps in sequence: `node scripts/build-search-index.js` (writes `public/search-index.json`) then `astro build`. Do not skip the first step or search will be broken.

## Database

### Schema

```sql
CREATE TABLE tartans (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  name        TEXT NOT NULL,
  description TEXT DEFAULT '',
  palette     TEXT NOT NULL,
  threadcount TEXT NOT NULL,
  slug        TEXT UNIQUE NOT NULL,
  parent_slug TEXT,                    -- set for user-created variants
  is_official BOOLEAN DEFAULT 0,       -- 1 for Scottish Register tartans
  origin_url  TEXT DEFAULT '',
  created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_slug    ON tartans(slug);
CREATE INDEX idx_letter  ON tartans(SUBSTR(slug, 1, 1));
CREATE INDEX idx_official ON tartans(is_official);
```

Migrations live in `migrations/`. Run new migrations with:
```bash
npx wrangler d1 migrations apply gordon-tartan-db --local   # local
npx wrangler d1 migrations apply gordon-tartan-db --remote  # production
```

### Accessing D1 in Pages/API Routes

```ts
const db = Astro.locals.runtime.env.DB;  // Astro pages
const db = locals.runtime.env.DB;        // API routes
```

The binding name `DB` is defined in `wrangler.toml`. The `D1Database` type is imported at the bottom of `src/lib/db.ts` via:
```ts
type D1Database = import('@cloudflare/workers-types').D1Database;
```

### Seeding

The seed script (`scripts/seed-d1.js`) reads `src/data/tartans.csv`, generates unique slugs (appending a numeric suffix for duplicates), and batches INSERT statements in groups of 50 to avoid D1 limits.

## Core Library (`src/lib/`)

### `tartan.ts` вЂ” Pattern Logic

Key exports:

| Function | Purpose |
|---|---|
| `slugify(str)` | Converts a tartan name to a URL-safe slug. Strips accents, replaces spaces with hyphens, strips non-word chars. |
| `countPattern(threadcount, palette)` | Parses threadcount + palette into an array of `{fill, size}` entries for SVG rendering. Handles symmetric (pivot `/`) and repeating patterns. |
| `buildSvgString(threadcount, palette)` | Returns `{svg, size}` вЂ” a full SVG string with horizontal + vertical stripe groups and a crosshatch grating mask. |
| `svgToDataUri(svg)` | Encodes SVG as a `data:image/svg+xml;utf8,...` URI. |
| `hexToHue(H)` | Converts a hex color to a hue value (0вЂ“359) for sorting. |
| `sortPaletteColors(palette)` | Returns palette colors sorted by hue. |

**Tartan data formats:**

- **Palette**: space-separated `CODE#RRGGBB` entries, e.g. `G#005020 K#101010 W#E0E0E0`
- **Threadcount**: space-separated `CODE[/]N` entries, e.g. `G/36 K2 W4 K2 G/36`
  - A `/` after the code marks a **pivot point** (symmetric pattern)
  - If the first entry has `/`, the pattern is symmetric (mirrors at pivots)
  - Without `/`, the pattern simply repeats
  - `size` in `ThreadEntry` = `4 Г— thread count` (pixel scaling factor)

### `db.ts` вЂ” Database Access

All functions are `async` and accept a `D1Database` as first argument.

| Function | Description |
|---|---|
| `getTartanBySlug(db, slug)` | Fetch one tartan by slug or `null` |
| `getTartansByLetter(db, letter, page, pageSize)` | Paginated listing for a letter; default page size 60 |
| `getAdjacentTartans(db, slug)` | Previous and next tartans by slug sort order |
| `getRandomOfficialSlug(db)` | Random slug from `is_official = 1` tartans |
| `createTartan(db, data)` | Insert a new tartan row; returns the created record |
| `getLetterPageCount(db, letter, pageSize)` | Total page count for a letter |

Queries use parameterised binds (`prepare(...).bind(...)`) throughout вЂ” never string interpolation.

## React Components (Islands)

React components are used only where interactivity is required. They are mounted with Astro's island directives:

- `client:load` вЂ” hydrate immediately on page load (`TartanEditor`)
- `client:idle` вЂ” hydrate when browser is idle (`SearchWidget` in `Layout.astro`)
- `client:visible` вЂ” hydrate when visible in viewport (`PngDownloadLink`)

### `TartanEditor.tsx`

Full-screen editor for creating custom variants. Contains a local copy of `countPattern` for the live SVG preview (intentional duplication to avoid bundling issues). Saves via `POST /api/tartans` and redirects to the new tartan page on success.

### `SearchWidget.tsx`

Lazy-loads `/search-index.json` on mount. Uses Lunr.js with fuzzy matching (edit distance 1 for terms > 5 chars) and wildcard matching. ANDs multiple search terms together.

## Testing

Tests live alongside source in `src/lib/`:

```bash
npm test             # Run all tests
npm run test:watch   # Watch mode
```

### Test conventions

- Vitest with `environment: 'node'`
- Each function has a `describe` block; each case is a short `it` with a descriptive label
- D1 is mocked in `db.test.ts` using a queue-based helper (`makeDb`) вЂ” callers push ordered async factory functions, which are consumed in sequence by `first()`, `all()`, and `run()` calls
- Only `src/lib/` is currently unit-tested; page/component testing is not set up

### Adding tests

When adding new functions to `tartan.ts` or `db.ts`, add corresponding tests in the matching `.test.ts` file. For new `db.ts` functions, extend the `makeDb` mock pattern.

## Key Conventions

1. **No ORM** вЂ” all DB queries are raw SQL via D1's `prepare().bind().first/all/run()`
2. **Slug uniqueness** вЂ” slugs are generated with `slugify(name)` and deduplicated by appending `-2`, `-3`, etc.
3. **Pagination** вЂ” 60 items per page across the A-Z listing; the `page` segment is optional (defaults to 1)
4. **SVG generation is server-side** for the background tile (`SvgBg.astro`) and download links; the editor preview is client-side React
5. **`slugify` is duplicated** вЂ” `src/lib/tartan.ts` (TypeScript) and `scripts/` (plain JS). Keep them in sync if the logic changes
6. **CSS** вЂ” all styles in `src/styles/global.css`; CSS custom properties defined in `:root` (e.g. `--primary`, `--accent`, `--white`, `--black`)
7. **No image optimisation** вЂ” `astro/assets/services/noop` is configured because `sharp` is not available in Cloudflare Workers
8. **`nodejs_compat`** flag is required in `wrangler.toml` for Node.js APIs in the worker bundle

## Environment / Secrets

No `.env` file is required for local dev вЂ” Wrangler's `platformProxy` exposes the local D1 binding. For production, the D1 database ID is hardcoded in `wrangler.toml`:

```toml
database_id = "e0e62aa5-5586-4370-b7d9-e960a8a31437"
```

There are no API keys or secrets in this project.

## Common Tasks

### Add a new page
Create a `.astro` file under `src/pages/`. Access D1 via `Astro.locals.runtime.env.DB`. Use `Layout.astro` as the wrapper.

### Add a new DB query
1. Add the function to `src/lib/db.ts` (typed, parameterised)
2. Add tests in `src/lib/db.test.ts` using the `makeDb` mock
3. Import and call from the relevant page

### Add a new interactive component
1. Create a `.tsx` file in `src/components/`
2. Import in the `.astro` file and add a `client:*` directive
3. Keep component-internal logic separate from `src/lib/` utilities

### Update tartan data
1. Edit `src/data/tartans.csv`
2. Re-run `npm run seed -- --local` to refresh the local DB
3. Re-run `npm run build` to regenerate the search index

### Run a manual DB query locally
```bash
npx wrangler d1 execute gordon-tartan-db --local --command="SELECT COUNT(*) FROM tartans"
```
